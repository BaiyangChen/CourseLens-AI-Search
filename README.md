# CourseLens AI Search
CourseLens AI Search is a prototype system for analyzing educational videos and recommending tutorials based on course content and student learning preferences.
The system extracts keyframes from tutorial videos, analyzes them with a local vision-language model, generates structured course profiles, and uses these profiles to recommend suitable courses based on a user's natural language query.

## Main Features

- Extract keyframes from educational videos
- Analyze keyframes with a local vision-language model
- Generate structured course profiles
- Match user queries with existing course topics
- Recommend tutorials based on topic relevance and learning preferences
- Provide explainable results with matched topics, reasons, and warnings
- Run a simple benchmark for search performance

## Prerequisite
- Python 3.12 or compatible version
- Ollama
- A web browser
- Local Ollama models:qwen3-vl:4b for keyframe analysis
- phi4-mini:3.8b for query expansion

## Install the Ollama models:
```
ollama pull qwen3-vl:4b
ollama pull phi4-mini:3.8b
```

## Installation
- Clone the repository:
```
git clone https://github.com/BaiyangChen/-CourseLens-AI-Search.git
cd -CourseLens-AI-Search
```
- Create and activate a virtual environment:
```
python -m venv .venv
.venv\Scripts\activate
```
- Install Python dependencies:
```
pip install -r requirement.txt
```

## Run the Search Demo
- The search demo uses the existing course profiles stored in:
```
output/analysis/course_profile.json
```
- Start the local API server:
```
python -m course_analyzer.front_end.search_api
```
- Open the browser:
```
http://127.0.0.1:8000
```
- Enter a natural language query, select learning preferences, and click Search.
  -Example query: I want to follow a complete tutorial that builds a small web application with frontend, backend, API calls, and a database.

- Example preferences: Hands-on coding, Project-based

### The system returns ranked tutorials with match scores, matched topics, reasons, and warnings.

## Run Video Analysis with CLI
- The cli.py script runs the offline video analysis pipeline for the two videos in the data folder:
```
data/Branching_Iteration.mp4
data/Chinese_Hinstory.mp4
```
Run:
```
python -m course_analyzer.cli
```
- This process will:
  1. Extract keyframes from both tutorial videos
  2. Save frames to output/frame1 and output/frame2
  3. Analyze each frame with qwen3-vl:4b
  4. Save frame analysis results
  5. Generate a new course profile file: output/analysis/course_profile2.json

### Note: this step can take time because it calls a local vision-language model for every extracted keyframe.

## Run Benchmark
### The benchmark evaluates the search and recommendation stage, not the full video analysis stage.
- Run the API server first:
```
python -m course_analyzer.front_end.search_api
```
- Then run:
```
python test\benchmark_search.py
```
- The benchmark reports:
```
Précision des réponses
Temps de réponse moyen
Taux d'automatisation
```
- Example result:
```
Précision des réponses: 100.0%
Temps de réponse moyen: 5.52 seconds
Taux d'automatisation: 100.0%
```

## Notes
### The default search API currently reads: output/analysis/course_profile.json
### If you want the search demo to use the profiles generated by cli.py, replace course_profile.json with course_profile2.json, or update COURSE_PROFILE_PATH in course_analyzer/front_end/search_api.py.
### Large local video files, virtual environments, generated frames, and cache files should not be committed to GitHub.
